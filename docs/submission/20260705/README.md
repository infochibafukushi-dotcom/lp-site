### PDF版とWord版の使い分け

- **PDF版**（`pre-fixed-fare-submission-full-set-v1-candidate.pdf` 等）は、提出候補版として維持する。運輸局への説明・印刷提出の基準資料とする。
- **Word版**（`.docx`）は、PDFの代替ではなく、編集・手動調整用の追加ファイルである。表・本文をWord上で修正してから再印刷・再PDF化する場合に使用する。
- Word版はPDFを直接変換したものではない。`data/*.json` および `shared/*` の元データから、編集可能なテキスト・表・画面証跡画像として生成する。

### 印刷方法

1. 管理画面の「提出セット一括PDFを開く」、または `pre-fixed-fare-submission-full-set-v1-candidate.pdf` をブラウザで開く
2. ブラウザの印刷画面を開く（Ctrl+P / ⌘+P）
3. 用紙サイズを **A4** に設定する
4. 部数を **3部** に指定する（関東運輸局手引きの正・副・控に相当）
5. 印刷またはPDF保存を実行する

※ 3部ぶん同じ内容をPDF内で3回連結する方式は採用していません。印刷ダイアログの部数指定で3部印刷してください。

※ 個別PDF 6件は引き続き本フォルダに保管してください。一式PDFは印刷用の結合版です。final化は正式提出日前に別途行います。

### 読みやすさ改善（2026/07/05）

運輸局への紙提出・説明を想定し、**ページ数削減より崩れないレイアウトを優先**する印刷用基準を適用しています。

- フォントサイズ基準：申請書タイトル 15pt / h1 18pt / h2 16pt / h3 12.5pt / 本文 10.5pt / 表 9〜9.5pt / 注記 8.5pt
- ページ余白：`@page margin 8mm`（右端切れ防止のため固定幅720pxは不使用）
- 章開始のみ改ページ。別紙1は新ページ先頭から開始
- 平準化係数表・別紙1距離制運賃表は `no-orphan-table` でヘッダー孤立を防止
- 候補版一式PDFの別紙セットは記入補助・平準化係数・別紙1・**別紙2（各種料金表）**まで
- **別紙2**は `data/estimate-config.json` および `estimate/estimate-calc.js` の設定値・計算ロジックから金額を確認して作成（推測入力なし）
- 未設定・未使用項目（予約料、キャンセル料等）は「設定なし」「実費」「精算時確定」として整理
- 事前確定運賃と各種料金（迎車料・介助料・待機料等）は区分して別行表示
- 表セル余白：`padding 2.2mm`（上下）、`line-height 1.45`
- 白紙ページ検出（ink ratio）を再生成時に実施
- 一式PDFの総ページ数（最新再生成時: **48ページ**）
